最近接触了不少新的东西，对自己职业规划和技术发展有了一些新的想法。

很幸运，有位靠谱的师兄帮我看了简历，还转发了一位后端大佬给我提的不少十分中肯的意见。

我是2017年初才决定要走后端路线的。大一到大三，没有把主要的经历放在技术上，现在有点后悔。基础专业课 也没有学扎实。当然我对技术还是有一点野心的。编程的乐趣是其它事情无法替代的，就像写作和阅读一样。

事实上大学走了不少弯路，大一学 C/C++ 入门，也止于入门。大二接触了 Java，大三在某两门课的大作业用 Java 做网络编程和 安卓 App。但没有找到对的方法，不仅没有把项目做好，也磨灭了兴趣。不过这也让我排除了几个选项，我不适合做 GUI 或前端开发， 我还是希望能专注于逻辑的设计，并不擅长做出美观的界面和合理的交互设计。也不是说不擅长，是以我完美主义的性格，在界面上我 可能会追求极致以至于没法用平常心完成日常工作。

我觉得我的兴趣在于，基础架构、网络协议这些用户不可见的领域。

大四的时候找了一份实习，做 Web 后端开发，从零开始学 Golang。从此走上后端之路。现在基本把精力放在 Golang的 学习上面。会把精力放在 Golang 上是因为，它对高并发的支持

网络、数据库、Linux、 负载均衡、数据一致性
